// OPERATIONS

The invisible work that kills velocity — now automated

Ops is every task that falls on "whoever has bandwidth" — hiring, onboarding, vendor management, compliance, IT. None of it grows the business; all of it will kill you if ignored. A small operations agent stack handles the drudgery and flags the exceptions for humans.

hiring onboarding SOC2 vendor mgmt IT
01

Recruiting pipeline agent

From JD to offer, the agent handles the plumbing.

  • JD generator — writes role descriptions from a 1-line ask + the company context pack.
  • Sourcing agent — screens LinkedIn/GitHub matches against the JD rubric, produces a shortlist with evidence per candidate.
  • Scheduling agent — handles the 4-email back-and-forth to book screens.
  • Interview-kit generator — creates a tailored rubric + 5 questions per interviewer based on the candidate's background.
Never let the agent make the hire/no-hire call or auto-reject. Screening bias, fair-hiring laws, and reputational risk all say: humans decide.
02

Interview debrief synthesizer

After each interview, the agent consumes the scorecard, pulls quotes, and produces a hire-committee packet.

  • Structured summary against the rubric.
  • Highlighted disagreements between interviewers (the most useful thing to surface).
  • Flags any evidence gaps before the decision meeting.
03

Onboarding conductor

Day 1 through Day 30 should run itself.

  • Pre-start: orders laptop, creates accounts (Google, Slack, GitHub, 1Password), sends welcome email, schedules intros.
  • Day 1: a personalized Notion page with their first-week plan, owner assignments, reading list.
  • Weekly: check-in survey; if sentiment dips, escalates to the manager.
  • Day 30: generates a structured "here's what they've learned, here's where they're stuck" doc for the manager to review.
Example: see examples/operations.md — a filled-in onboarding runbook (T-7 through Day 30) for a fictional startup.
04

Compliance & evidence agent (SOC2, ISO, HIPAA)

Most early-stage companies procrastinate on compliance until a customer blocks a deal. Instead, start collecting evidence from day one with an agent.

  • Connects to Drata/Vanta/Secureframe via API; pulls failing control list weekly.
  • For each failure, it drafts the remediation: a ticket, a policy doc, or a process change.
  • Runs quarterly access reviews: lists who has access to what, who shouldn't, and drafts the revocation PRs.
05

Vendor & SaaS agent

Startups accumulate 40+ SaaS subscriptions by Series A. An agent keeps them sane.

  • Pulls the SaaS list from your finance system + SSO logs.
  • Flags: unused seats, overlapping tools, auto-renewals <30 days out, MSAs that haven't had a security review.
  • Drafts cancellation emails, negotiation asks, and consolidation proposals.
# Negotiation draft (renewal)
Our current plan is $X for Y seats; we use Z seats actively (60% utilization).
We'd like to renew at Z seats at a 15% discount given the utilization gap,
with a 12-month term. If that's not possible, we're exploring <alt tool>.
06

IT & device agent

MDM events are the ops equivalent of a monitoring alert. Treat them that way.

  • New device enrolled → checks against the hiring roster (was this expected?).
  • Disk unencrypted, OS out of date, suspicious app installed → drafts remediation steps.
  • Offboarding: a single command revokes SSO, wipes device, forwards email, transfers Drive ownership.
07

Policy & playbook library

Every repeated ops decision should have a policy. Every policy should be readable by agents.

  • One markdown file per policy: expense, travel, remote work, PTO, paternity, on-call.
  • An "HR assistant" agent answers teammate questions by citing the policy.
  • If no policy exists, it says so and routes to a human to author one — which is the whole point.
08

Ritual: the weekly ops sweep

30 minutes, every Friday. The agent dumps its working memory into one doc and you decide what matters.

  • Hiring: open roles, aging candidates, offers out.
  • Onboarding: people in flight, sentiment flags.
  • Compliance: failing controls, upcoming audits.
  • Vendors: renewals this month, unused seats.
  • People: 1:1s missed, tenure anniversaries, birthdays.
Principle: automate the clerical, not the judgmental. Agents route, draft, remind, and summarize. Hiring decisions, firing decisions, comp decisions, policy decisions — humans.
