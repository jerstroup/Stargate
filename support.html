// CUSTOMER SUPPORT

Two humans that feel like twenty

Agentic support isn't about replacing humans with a chatbot. It's about wrapping your two best teammates in an agentic harness: the AI triages, drafts, summarizes, and routes — humans spend 100% of their time on the 10% of tickets that matter, and deflection happens honestly.

triage deflection call summaries bug feedback loop
01

Build the knowledge base first (or nothing else works)

A support agent is only as good as its knowledge base. Most startups skip this and wonder why the bot lies.

  • Structured docs in git — one markdown file per topic. Not a sprawling Notion maze.
  • Include: product docs, known issues, workarounds, pricing, refund policy, SLAs.
  • Every doc has a frontmatter: owner, last-verified date, scope (self-serve? paid tier only?).
  • CI job: flag any doc not verified in 90 days. Dead docs cause hallucination.
Example: see examples/support.md — a filled-in KB article with frontmatter for a fictional startup.
02

Triage agent (every inbound ticket)

Classifies, tags, priorities, and routes. Runs before a human sees the ticket.

  • Labels: bug, how-to, billing, feature-request, churn-risk, security.
  • Pulls customer context (plan, ARR, last 3 tickets) and attaches it.
  • Auto-escalates churn-risk and security to a human within 5 minutes, regardless of hours.
03

Draft-response agent (human-reviewed)

For how-to and billing tickets, an agent drafts the response using the KB + the customer's history. Human clicks "send" or edits.

  • Constraint: agent must cite the KB doc it used. No cited doc = don't send.
  • Voice match to brand spec. Support voice = warmer than marketing voice; keep a separate section in voice.md.
  • For billing questions it can propose, not execute. Refunds always require a human.
Never let the agent make a promise in writing. Deliverability dates, SLA commitments, and refund amounts come from humans. The agent can surface options; humans commit.
04

Self-serve deflection, honestly done

A chat widget that answers self-serve questions with citations and offers "talk to a human" on every message.

  • Confidence threshold. If the agent's confidence is low, it opens a ticket instead of guessing.
  • Every deflected conversation is logged and sampled weekly — 20 random conversations read by a human to catch regressions.
  • "Was this helpful?" at the end, with the answer feeding the eval set.
05

Call-summary & follow-up agent

For phone/video support calls: auto-record (with consent), transcribe, summarize, extract commitments, create follow-up tasks.

  • Summary template: context, issue, resolution attempted, outcome, next step, owner, due date.
  • Posts summary back into the ticket within 60 seconds of call ending.
  • Drafts the follow-up email referencing commitments made on the call.
06

Bug-to-engineering feedback loop (the highest-ROI agent)

When a ticket is labeled bug, a specialized agent builds an engineering-ready issue and files it in Linear.

  • Extracts: repro steps, browser/OS/app version, affected account id, logs from the error-tracking system (Sentry via MCP).
  • Searches existing issues to dedupe.
  • If it matches an existing issue, adds the customer to the "notify-when-fixed" list automatically.

07

Voice-of-customer digest

Weekly agent-generated digest to the whole team. Crucially, not just "top complaints" — top shifts.

  • What new complaint theme emerged this week vs last month?
  • Which feature requests are from high-ARR accounts?
  • Which existing bugs are regressing in frequency?
  • Routed into the PM opportunity-scoring agent (see Product).
08

Measurement that reflects reality

  • First-response time — for AI draft, target <1 min; for human-sent, target <1 hr.
  • Deflection quality — % of deflected conversations where the customer didn't come back within 48 hr with the same issue.
  • Escalation accuracy — % of agent-escalated tickets that humans agreed should have been escalated.
  • Bug-to-fix latency — customer-reported bug → engineering issue filed → PR merged. Target <72hr for P1.
