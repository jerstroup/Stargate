// FINANCE

Models investors trust, built by two people and an agent

You don't need a CFO at seed. You need a disciplined financial system: one live model, one clean metric definitions file, and a finance agent that refreshes both automatically. Investors don't care about perfect models — they care about a founder who knows their numbers cold. This stack makes that cheap.

financial model scenario planning investor metrics cash runway
01

Build the metric dictionary first

A YAML/markdown file that defines every metric precisely. If ARR means three things to three people, your model lies.

  • ARR — what's in, what's out (trials, credits, one-time fees)?
  • Gross margin — what COGS? (hosting, payment fees, support).
  • Burn — gross vs net. Pick one, label it.
  • NRR, CAC, payback, magic number, rule of 40 — each with the SQL that computes it.
This file is the interface between your data warehouse, your financial model, and every investor update. Version it.
02

Pick the model format: spreadsheet-as-data

Google Sheets or Causal, not Excel. You want an API. You want diffs.

  • Structure: inputs tab, drivers tab, P&L, cashflow, cap table, scenarios.
  • Every formula references named ranges, not cell addresses.
  • The model exports JSON every morning to a location the agents can read.
03

The Finance Agent

One agent, three daily jobs.

  • Actuals sync — pulls from Stripe, Mercury, Brex, Gusto via MCP; updates the actuals tab; flags anomalies (>20% variance vs forecast).
  • Forecast refresh — re-runs drivers against updated actuals; updates runway, burn multiple, and cash-out date.
  • Narrative draft — produces a 1-page monthly financial narrative: top 3 moves, runway update, decisions that need to be made.
# Narrative draft prompt
Using actuals.json and forecast.json, write a 1-page monthly finance memo.
Sections:
  - Headline (1 sentence: are we on track?)
  - Revenue: MRR, growth %, top 3 account moves
  - Cash: burn, runway, variance vs plan
  - Decisions needed (max 3, each with a clear ask)
No adjectives like "robust" or "strong". Numbers or nothing.
04

Scenario agent (the hire/pause/raise model)

The CEO should be able to ask "what if we hire 3 engineers and close the pipeline at 60%?" and get an answer in 30 seconds.

  • Agent takes natural-language scenarios, translates to driver changes, runs the model, returns the 4-number summary: new runway, new cash-out, new ARR-at-raise, new implied valuation.
  • Saves every scenario with a timestamp so you can show an investor "here's the sensitivity."
05

Investor update generator

Monthly investor updates should be a 20-minute job, not a 3-hour slog.

  • Pulls metrics, narrative, top wins, top losses, hiring asks.
  • Uses a template with five sections: Headline, KPIs, Highlights, Lowlights, Asks.
  • Sends draft to the founder; founder edits the story, facts are already correct.
Investors read the Asks section first. Put real asks — intros, hiring, advice on a decision. Not "nothing, we're good."
06

Focus on what investors actually price on

Most finance hallucination comes from founders modeling the wrong things. Anchor on these:

StageNumbers that price
Pre-seedFounder quality, design partners committed, weekly shipping cadence
SeedMonth-over-month revenue growth, first 5 logos, gross retention signals
Series AT2D3 trajectory, >100% NRR, CAC payback <24mo, rule of 40 trending right
Series B+Magic number >0.7, gross margin >70%, cohort retention curves flattening
Everything below Series A: stop modeling 5 years out. It is a lie and investors know it. Model 18 months of runway + a 6-month plan to the next milestone.
07

Bookkeeping + compliance on autopilot

  • Classification agent — categorizes transactions in QuickBooks/Xero nightly; flags the ones it can't confidently label.
  • Invoice agent — reads inbound vendor invoices from email, matches POs, files them.
  • Tax / filings agent — tracks franchise tax, sales-tax nexus (per state), Delaware annual report, 83(b) windows. Pings calendar 30 days before each.
08

Fraud & anomaly watchdog

An agent that compares today's spend + revenue to the last 30 days and raises a Slack alert on anything >3σ.

  • Ghost charges (double-billed SaaS).
  • Sudden refund spikes (support issue upstream).
  • Card used in a new geography (stolen card).
  • Payroll change you didn't approve.
Rhythm: daily actuals sync, weekly 15-min metrics review with founder, monthly investor update, quarterly board pack. All drafted by agents, decided by humans.
